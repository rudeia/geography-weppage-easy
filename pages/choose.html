어떤 방법으로 시작하면 좋을까요?

이 페이지는 도구를 하나로 정해주는 곳이 아닙니다. 수업 목적에 맞게 제작 방식을 고르도록 돕는 지도입니다. 먼저 만들고 싶은 결과를 고른 뒤, 필요한 경우 프로그램별 상세 가이드로 이동하세요. 어떤 프로그램이 “정답”인 것은 아닙니다. 선생님의 아이디어를 가장 덜 복잡하게 실현하는 길을 찾는 것이 목표입니다.

초보자 추천 경로 고르기

처음부터 모든 프로그램을 비교하면 어렵습니다. 먼저 아래 세 가지 길 중 하나를 고른 뒤, 필요한 도구만 확인하세요.

경로 1

기존 앱 유지형

이미 쓰고 있는 수업 플랫폼으로 충분하면 새 웹페이지를 만들지 않습니다. 이 길은 “덜 만드는 것”이 아니라, 수업 운영을 가장 단순하게 유지하는 선택입니다.

  • 공지·과제·성적 흐름: Classroom, 학교 LMS, Teams
  • 설문·신청·피드백: Google Forms, Microsoft Forms, Tally
  • 퀴즈 리포트: Kahoot, Quizizz, Wordwall
  • 협업 게시판: Padlet, Google Docs, FigJam
  • 문서형 자료 허브: Notion, Drive 폴더
이 경로가 맞는 경우 학생이 이미 익숙한 앱에서 과제 제출과 피드백까지 잘 이어진다면 이 경로가 가장 효율적입니다.
경로 2

노코드 안내형

코드 파일을 만들지 않고, 서비스 화면 안에서 안내 페이지나 활동을 만듭니다. 수업 안내, 자료 모음, 간단한 조작 활동, 제출 링크 연결에 적합합니다.

  • 보기 좋은 안내: Canva AI, Canva 웹페이지
  • Google 자료 허브: Google Sites
  • 긴 안내·자료 모음: Notion
  • 눌러보는 활동: Genially
  • 문서형 설문과 신청: Tally
이 경로가 맞는 경우 학생이 한 주소에서 안내를 읽고, 버튼을 누르고, 제출 링크로 이동하면 충분할 때 좋습니다.
경로 3

HTML 실험형

AI가 만든 `index.html` 파일을 웹에 올려 직접 만든 화면을 공유합니다. 기존 앱으로 만들기 어려운 맞춤형 화면이나 작은 상호작용을 실험할 때 선택합니다.

  • 첫 파일 만들기: ChatGPT, Codex, Claude 계열 AI
  • 첫 배포: Netlify
  • 장기 관리: GitHub Pages
  • 자동 배포와 운영: Cloudflare Pages
이 경로가 맞는 경우 “이런 화면이 있었으면 좋겠다”가 뚜렷하고, 기존 앱으로는 그 화면을 만들기 어려울 때 적합합니다.
처음이라면 추천 순서 기존 앱으로 가능한지 먼저 봅니다. 부족하면 노코드 안내형으로 작게 만듭니다. 그래도 필요한 화면이 안 나오면 HTML 실험형으로 넘어가세요. 이 순서를 지키면 웹페이지 제작이 수업보다 앞서는 상황을 줄일 수 있습니다.

수업 장면으로 보면 더 쉽습니다

아래 예시는 “이럴 때는 어떤 길이 자연스러운가?”를 보여주는 작은 판단 연습입니다. 내 수업이 어느 예시에 가까운지 먼저 고르면 도구 선택이 훨씬 쉬워집니다.

수행평가 과제 제출추천 경로: 기존 앱 유지형학생 명단, 제출 여부, 피드백, 성적 흐름이 중요하다면Classroom이나 학교 LMS를 유지하는 편이 좋습니다.웹페이지는 평가 기준과 예시 자료를 보여주는 보조 안내판 정도로 충분합니다.추천 수준 연결: 직접 제작보다 기존 앱을 먼저 봅니다.안내판이 필요할 때만 `처음 시작 추천` 도구를 확인하세요.
답사·행사 안내추천 경로: 노코드 안내형일정, 장소, 준비물, 지도 링크, 신청 링크를 한 화면에 모으는 목적이라면Google Sites나 Canva AI가 적합합니다.학생 응답은 Forms나 Tally로 연결하는 편이 안전합니다.안내 화면 자체에는 개인정보를 직접 적지 않는 것이 좋습니다.추천 수준 연결: Canva AI, Google Sites, Tally부터 확인합니다.
기후 그래프 판별 활동추천 경로: HTML 실험형학생이 그래프를 고르고 버튼을 누른 뒤, 결과 설명을 바로 확인해야 한다면AI가 만든 HTML/JavaScript 활동이 의미가 있습니다.첫 배포는 Netlify로 작게 성공해봅니다.계속 수정할 자료라면 GitHub Pages나 Cloudflare Pages로 확장합니다.추천 수준 연결: Netlify로 먼저 성공한 뒤 장기 운영 도구를 봅니다.
추가 수업 예시 더 보기 전시, 방탈출, 대시보드, 모둠 체크 앱을 비교합니다.
자료 전시회추천 경로: 노코드 안내형학생 작품, 발표 자료, 탐구 결과를 모아 보여주는 목적이라면Google Sites, Canva 웹페이지, Notion이 적합합니다.댓글, 점수, 제출 확인이 필요하면 Forms나 Tally를 별도로 연결합니다.추천 수준 연결: `처음 시작 추천` 도구만으로 충분한 경우가 많습니다.
방탈출형 수업추천 경로: 노코드 안내형 또는 HTML 실험형이미지 위 팝업, 단계별 단서, 숨겨진 설명은 Genially로 시작하기 좋습니다.정답 판정, 점수 계산, 조건에 따른 다음 화면 이동이 필요하면AI가 만든 HTML/JavaScript를 검토합니다.추천 수준 연결: Genially는 `상황 맞으면 추천`입니다.직접 만든 HTML 활동은 배포 도구와 함께 봅니다.
학습자 분석 대시보드추천 경로: 기존 앱 유지형에서 시작 후 필요 시 확장설문 결과를 보기만 하면 Forms와 Sheets로 충분할 수 있습니다.모둠 구성, 관심 주제, 활동 결과를 계속 관리하려면AppSheet, Canva Sheets, Supabase를 비교합니다.추천 수준 연결: AppSheet는 `상황 맞으면 추천`입니다.Supabase와 Firebase는 `고급 단계`로 봅니다.
모둠 활동 체크 앱추천 경로: 노코드 안내형 또는 응답 관리형체크리스트와 제출 현황만 필요하면 Google Sheets나 AppSheet가 현실적입니다.학생이 직접 입력하고 교사가 상태를 계속 확인해야 한다면AppSheet를 먼저 검토합니다.추천 수준 연결: 단순 제출은 Forms나 Tally가 더 쉽습니다.

응답·개인정보 먼저 점검하기

도구를 고르기 전에 학생에게 무엇을 입력하게 할지 먼저 정해야 합니다. 입력을 받지 않는 안내 페이지와, 학생 응답을 저장하는 웹앱은 위험도와 준비물이 다릅니다.

받지 않아도 되는 정보는 빼기

이름, 학번, 얼굴 사진, 성적, 연락처는 꼭 필요한 경우가 아니면 받지 않습니다. 별칭, 모둠, 번호처럼 줄일 수 있는지 먼저 봅니다.

저장 위치 확인하기

Google Forms, Tally, AppSheet, Supabase, Firebase 중 어디에 응답이 저장되는지 확인합니다. 저장 위치를 모르면 공개 전에 멈추는 것이 안전합니다.

공개 범위 나누기

학생이 보는 화면, 교사만 보는 응답, 학급 전체에 공유할 요약 결과를 나눕니다. 원자료를 그대로 공개하지 않는 것이 기본입니다.

먼저 묻는 질문 학생 응답이 필요 없으면 안내·배포 도구부터 보면 됩니다. 예: Canva AI, Google Sites, Notion, Netlify 응답이 필요하면 Forms나 Tally처럼 안정적인 제출함부터 검토하세요. 권한 관리가 필요한 경우에만 AppSheet, Supabase, Firebase로 확장합니다.

내 제작 방식 고르기

처음 만드는 교사라면 프로그램 이름부터 고르지 않아도 됩니다. 먼저 “학생에게 안내판이 필요한가, 눌러보는 활동이 필요한가, 응답을 모아야 하는가”를 떠올립니다. 그다음에 알맞은 도구를 고르는 편이 훨씬 덜 막막합니다.

이 페이지에서 고르는 순서 수업자료 제작은 한 줄 사다리가 아니라 여러 갈래 길입니다. 먼저 필요한 기능을 고른 뒤, 그 기능을 가장 쉽게 해주는 도구를 고릅니다. 안내만 필요하면 Canva AI나 Google Sites부터 봅니다. 학생이 눌러보는 활동은 Canva AI나 Genially가 출발점입니다. 응답 수집은 Forms나 Tally부터 시작합니다. AI가 만든 `index.html` 파일을 직접 배포해야 할 때만 파일 구조, Netlify, GitHub Pages, Cloudflare Pages를 차례대로 확인하세요.

가장 쉬운 시작: 안내 페이지 만들기

수업 안내, 수행평가 안내, 자료 링크 모음처럼 학생이 읽고 따라갈 화면을 만듭니다.

  • 추천 도구: Google Sites, Canva AI/Canva 웹페이지, Notion
  • 추천 상황: 학부모 안내, 행사 안내, 자료실, 작품 전시, 짧은 활동 안내
  • 먼저 확인: 이 단계가 정말 내 수업에 맞는지 제작 흐름에서 확인

활동형 시작: 눌러보는 활동 만들기

  • 추천 도구: Canva AI, Genially, AI가 만든 HTML/JavaScript
  • 추천 상황: OX 퀴즈, 플래시카드, 매칭 게임, 팝업 설명, 지도 탐색

학생 의견, 신청, 피드백, 활동 완료 여부를 받을 때는 저장 방식의 난이도를 먼저 나눕니다.

  • 가장 쉬운 입력 수집: Google Forms, Tally
  • 시트 기반 관리 앱: Google AppSheet

확장

확장형 시작: 웹앱처럼 확장하기

여러 화면, 진행률, 필터, 대시보드처럼 단순 안내 페이지를 넘어서는 기능이 필요할 때 검토합니다.

  1. React/Next.js 프로젝트라면 Vercel을 검토합니다.
  2. GitHub 자동 배포와 운영 관리는 Cloudflare Pages를 검토합니다.
  3. 데이터 저장이 필요해도 먼저 화면과 배포가 제대로 되는지 확인합니다.
이 방식 먼저 이해하기
데이터 저장

저장형 시작: 학생 응답과 기록 저장하기

직접 만든 웹앱에서 응답을 저장해야 할 수 있습니다. 저장한 응답을 다시 불러와야 할 때 검토하는 단계입니다.

  1. 표 기반 응답 저장은 Supabase를 검토합니다.
  2. 로그인, 실시간 동기화, 파일 저장은 Firebase를 검토합니다.
  3. 단순 응답은 Forms나 Tally로 충분한지 먼저 비교합니다.
  4. 학생 개인정보, 공개 범위, 보안 규칙을 반드시 확인합니다.
이 방식 먼저 이해하기
같은 프로그램도 수업 아이디어에 따라 달라집니다 Canva는 카드뉴스만 만드는 도구가 아니라 간단한 퀴즈와 웹페이지로 확장될 수 있습니다. Google Sites는 자료실에서 출발해 Apps Script와 연결된 제출 확인 허브가 될 수 있습니다. Netlify나 Cloudflare는 단순 배포 도구로 시작하지만, 직접 만든 상호작용 활동을 학생에게 공유하는 통로가 됩니다. 처음부터 큰 기능을 고르기보다 “내 수업에서 학생이 어떤 행동을 하게 만들고 싶은가?”를 먼저 정해보세요.

파일을 직접 다룰지 판단하기

이 페이지에서는 파일 구조를 자세히 배우기보다, 내 작업에 파일 이해가 필요한지 먼저 판단합니다. Canva, Google Sites처럼 자체 게시가 되는 도구는 파일을 직접 만지지 않아도 시작할 수 있습니다. AI가 만든 HTML을 배포하려면 파일과 폴더를 함께 다루는 방식으로 넘어가야 합니다.

파일을 몰라도 되는 경우

Canva 웹페이지나 Google Sites처럼 화면 안에서 만들고 바로 게시하는 도구는 처음부터 `index.html`을 몰라도 됩니다. 교사는 내용, 링크, 공개 범위에 더 집중하면 됩니다.

파일을 조금 알아야 하는 경우

AI가 만들어준 HTML 파일을 Netlify, GitHub Pages, Cloudflare Pages에 올릴 때는 `index.html`이 첫 화면이라는 점을 알아야 합니다. 이미지나 PDF는 함께 올려야 하는 자료함이라고 이해하면 됩니다.

파일 구조를 제대로 봐야 하는 경우

퀴즈, 지도, 결과 화면처럼 JavaScript가 들어가거나 이미지와 자료가 많아지면 파일 구조 이해가 필요합니다. 이때는 제작 흐름의 HTML 파일 기반 제작 설명으로 넘어가면 됩니다.

자세한 설명은 제작 흐름에서 확인하세요 이곳에서는 “파일을 다루는 방식이 필요한가?”만 판단하면 됩니다. `index.html`, CSS, JavaScript, assets 폴더의 의미와 AI에게 첫 파일을 요청하는 방법은 제작 흐름의 HTML 파일 기반 제작 이해하기에서 이어서 확인하세요.

배포 도구 차이 비교하기

AI로 index.html을 만들었다면 보통 세 가지 배포 방식을 고민합니다. 모두 정적 웹페이지를 공개할 수 있지만, 첫 배포 난이도와 장기 관리 방식이 다릅니다.

첫 배포 추천

Netlify

가장 빠르게 웹주소를 만들고 싶을 때 좋습니다. `index.html`이 들어 있는 폴더를 드래그앤드롭하면 바로 배포할 수 있습니다.

  • 좋은 경우: 처음 HTML을 배포해보는 교사
  • 좋은 경우: 학생에게 빨리 주소를 공유해야 할 때
  • 좋은 경우: GitHub가 아직 낯설 때
  • 주의: 수정할 때 다시 배포하거나 GitHub 연결을 해야 함
장기 관리 추천

GitHub Pages

수업 자료를 오래 보관하고 매년 수정해 재사용하고 싶을 때 좋습니다. GitHub 저장소에 파일을 올리고 Pages 기능을 켜는 방식입니다.

  • 좋은 경우: 수행평가 안내 페이지를 매년 수정할 때
  • 좋은 경우: 동료 교사와 자료를 공유할 때
  • 좋은 경우: 여러 HTML 페이지를 체계적으로 관리할 때
  • 주의: 저장소, commit, Pages 설정이 처음에는 낯설 수 있음
자동 배포 추천

Cloudflare Pages

GitHub 저장소와 연결해 자동 배포를 관리하고 싶을 때 좋습니다. 미리보기 배포, 이전 버전 확인, 커스텀 도메인 연결까지 함께 고려할 수 있습니다.

  • 좋은 경우: GitHub에 올린 자료를 수정할 때마다 자동으로 반영하고 싶을 때
  • 좋은 경우: 수업 사이트를 장기 운영하며 버전 관리를 하고 싶을 때
  • 좋은 경우: 나중에 도메인 연결이나 여러 배포 환경을 고려할 때
  • 주의: 첫 HTML 한 장을 올리는 경험은 Netlify가 더 쉽습니다.
비교 항목Netlify는 빠른 첫 배포에 강합니다.GitHub Pages는 저장소 기반 장기 관리에 좋습니다.Cloudflare Pages는 GitHub 자동 배포와 운영 관리에 강합니다.
첫 배포 난이도Netlify가 가장 쉽습니다.폴더를 끌어다 놓는 방식이라 초보자가 바로 결과를 볼 수 있습니다.
수정과 관리GitHub Pages와 Cloudflare Pages는 GitHub 저장소를 중심으로 관리합니다.수정 기록을 남기며 오래 운영하기 좋습니다.
자동 배포Cloudflare Pages는 GitHub와 연결해 수정 사항을 자동 반영하는 흐름이 강합니다.Netlify도 GitHub 연결이 가능하지만, 첫 경험은 수동 배포가 더 쉽습니다.
추천 흐름오늘 바로 공유는 Netlify를 먼저 검토합니다.매년 고쳐 쓸 수업 자료는 GitHub Pages가 좋습니다.GitHub 자동 배포와 장기 운영은 Cloudflare Pages를 검토하세요.
안전 점검어떤 배포 방식을 쓰든 학생 개인정보, 자료 링크 권한, 공개 범위는공개 전 점검에서 다시 확인해야 합니다.
초보자에게 가장 현실적인 순서 AI 활용 페이지에서 `index.html`을 만든 뒤 Netlify로 먼저 배포해보세요. 웹주소가 만들어지는 경험을 먼저 해봅니다. 계속 쓸 자료는 GitHub Pages나 Cloudflare Pages로 관리하는 흐름이 좋습니다.

프론트엔드 작업용 AI 선택하기

HTML 코드 작성, 오류 수정, 배포 과정 설명은 AI의 도움을 많이 받을 수 있습니다. 특히 파일 구조를 이해하고 여러 번 수정해야 하는 작업에서는 코딩 작업에 강한 AI를 고르는 것이 결과물에 영향을 줍니다.

1 처음 시작

일반 대화형 AI로 구조 잡기

ChatGPT나 Claude 웹 대화창만으로도 첫 HTML 초안은 만들 수 있습니다. 수업 목적, 대상, 활동 흐름, 넣을 링크를 설명하면 `index.html` 형태의 초안을 받을 수 있습니다.

  • 좋은 작업: 첫 화면 문구 만들기, 섹션 구성, 버튼 이름 정하기
  • 주의할 점: 파일 여러 개를 다루거나 오류를 고칠 때는 대화만으로 한계가 생길 수 있음
2 유료 구독자 추천

ChatGPT Codex 또는 Claude Code

유료 구독자라면 코딩 작업에 특화된 Codex나 Claude Code를 추천합니다. 이유는 HTML, CSS, JavaScript 파일을 함께 보고 수정하는 작업에 더 적합하기 때문입니다.

  • 좋은 작업: 여러 파일 수정, 디자인 반영, 오류 원인 찾기, 배포 전 점검
  • 추천 상황: 직접 만든 웹페이지를 계속 수정하고 관리하려는 교사
3 여러 AI 사용 가능

최신 코딩 강점 모델 우선 검토

여러 AI를 모두 사용할 수 있는 교원이라면 HTML과 프론트엔드 코드 작성에 강한 모델을 우선 검토할 수 있습니다. 예: 최신 Claude Opus 계열처럼 코딩 평가가 좋은 모델 모델 이름은 자주 바뀌므로 실제 사용 직전 공식 서비스 화면에서 최신 모델명을 확인하세요.

  • 좋은 작업: 복잡한 프론트엔드 구조 제안, 긴 코드 검토, 오류 수정 방향 제시
  • 작은 참고: 단순 문구 작성보다 HTML 코드, 파일 구조, 오류 수정 능력을 기준으로 고르는 것이 좋습니다.
추천의 기준 어떤 AI가 “가장 좋다”보다 중요한 것은 수업 목적을 정확히 설명하는 것입니다. 수정 요청을 구체적으로 하는 것도 중요합니다. 다만 HTML 코드 작성과 프론트엔드 오류 수정은 모델 성능과 코딩 작업 환경의 영향을 크게 받습니다.

Windows·Mac 작업 환경 준비하기

웹페이지 제작은 사이트 선택만으로 끝나지 않습니다. 내 컴퓨터에서 파일을 만들고, 확인하고, 다시 수정할 수 있는 작업 환경도 결과물의 품질에 영향을 줍니다.

W Windows

윈도우에서 준비하기

  1. 브라우저는 Chrome 또는 Edge를 준비합니다.
  2. 코드를 직접 고칠 가능성이 있다면 Visual Studio Code 또는 Cursor를 설치합니다.
  3. AI가 만든 `index.html`은 바탕화면보다 별도 폴더에 저장합니다. 예: `class-webpage`
  4. 파일을 더블클릭해 브라우저에서 먼저 확인한 뒤 Netlify나 GitHub Pages에 올립니다.
  5. GitHub Pages를 오래 쓸 예정이면 GitHub Desktop을 함께 쓰면 파일 관리가 쉬워집니다.
M Mac

맥에서 준비하기

  1. 브라우저는 Chrome 또는 Safari를 사용할 수 있습니다. 배포 전에는 가능하면 Chrome에서도 확인합니다.
  2. 코드 편집은 Visual Studio Code 또는 Cursor를 추천합니다.
  3. 작업 폴더를 만들고 `index.html`, 이미지, PDF 파일을 한곳에 모읍니다.
  4. Finder에서 `index.html`을 열어 화면을 확인하고, 모바일 크기에서도 글자가 읽히는지 봅니다.
  5. Codex나 Claude Code처럼 파일을 직접 다루는 도구를 쓸 경우작업 폴더를 명확히 지정해두면 수정 실수가 줄어듭니다.
초보자에게 가장 중요한 원칙은 하나입니다. 웹페이지 파일, 이미지, 활동지 PDF를 한 폴더 안에서 관리하면 배포 오류가 훨씬 줄어듭니다.
다음에 볼 페이지 도구의 방향을 정했다면, AI로 첫 `index.html`을 만들어보세요. 또는 실제 프로그램별 절차를 확인해보세요.
