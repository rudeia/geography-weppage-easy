공개 전 점검

공유하기 전에 꼭 확인해요

교육용 웹페이지는 학생과 학부모가 볼 수 있습니다. 그래서 기능보다 안전한 공개가 먼저입니다.

이 페이지 목차
1. 개인정보·공개 범위 점검하기 2. 수업 전 5분 점검표 3. 학생 기기·학생 계정으로 확인하기 4. 서비스 권한·자동화 점검하기 5. 고급 용어 미니 사전 6. 막혔을 때 확인하기

개인정보·공개 범위 점검하기

웹페이지를 공개한다는 것은 교실 문 앞에 안내판을 세우는 일과 비슷합니다. 누가 볼 수 있는지, 어떤 정보가 드러나는지, 학생 자료가 과하게 공개되지 않는지 먼저 확인해야 합니다.

제작 필요성

기존 앱으로 충분한지 확인

  • 공지·과제는 Classroom/LMS 검토
  • 설문·신청은 Forms/Tally 검토
  • 퀴즈는 Kahoot/Quizizz/Wordwall 검토
개인정보

학생 정보는 최소한으로

  • 이름, 학번, 얼굴 사진, 성적 공개 주의
  • 번호나 별칭 사용 권장
  • 민감한 자료는 학교 계정 제한 검토
저작권

자료 사용 조건 확인

  • 교과서 이미지, 문제집 문항, 뉴스 전문 사용 주의
  • 출처 표시와 링크 연결 권장
  • 학생 작품 공개 전 동의 확인
모바일

학생 화면에서 다시 보기

  • 글자가 충분히 큰지 확인
  • 버튼을 누르기 쉬운지 확인
  • 링크 이름이 명확한지 확인
공개 범위

전체 공개가 필요한지 판단

  • 전체 공개, 링크 공개, 학교 계정 제한 구분
  • 제출 링크와 자료 링크 권한 확인
  • 공유 후 수정 방법 확인
공개 직전 마지막 질문 이 자료가 Classroom, 학교 LMS, Forms, Padlet, Kahoot 같은 기존 앱으로 더 안전하게 운영될 수 있다면 굳이 공개 웹페이지로 만들지 않아도 됩니다. 웹페이지를 공개하는 이유가 “학생이 더 쉽게 따라오기 위해서”인지 확인하세요. 단지 “만들었으니 공개하려는 것”이라면 한 번 더 멈춰도 좋습니다.

수업 전 5분 점검표

수업 직전에 모든 설명을 다시 읽기 어렵다면 아래 여섯 가지만 확인하세요.

1. 학생 기기에서 열기

교사용 컴퓨터가 아니라 학생 휴대폰이나 학생 계정으로 실제 주소를 열어봅니다.

2. 첫 버튼 누르기

가장 중요한 버튼을 먼저 누릅니다. 제출 링크, 자료 링크, 활동 시작 버튼은 반드시 확인합니다.

3. 개인정보 지우기

이름, 학번, 얼굴 사진, 성적, 내부 메모가 화면에 남아 있지 않은지 확인합니다.

4. 자료 권한 확인

사이트는 열리는데 자료가 잠길 수 있습니다. Drive 문서나 Forms 권한을 따로 확인합니다.

5. 모바일 가독성 확인

글자가 너무 작거나 버튼이 붙어 있지 않은지 봅니다. 이미지가 화면을 가리지 않는지도 확인합니다.

6. 응답 저장 위치 확인

학생이 제출한 응답이 어디에 저장되는지 확인합니다. 누가 볼 수 있는지도 함께 확인합니다.

학생 기기·학생 계정으로 확인하기

교사용 컴퓨터에서 잘 보이는 것과 학생에게 실제로 열리는 것은 다를 수 있습니다. 공개 직전에는 “내 화면”이 아니라 “학생 화면”을 기준으로 확인해야 합니다.

모든 항목을 다 볼 필요는 없습니다 내가 실제로 사용한 서비스와 연결 자료만 확인하세요. 예를 들어 Google Sites와 Drive만 썼다면 Sites, Drive, 학생 기기 확인만 먼저 보면 충분합니다.
1. 교사 휴대폰으로 열기

완성된 주소를 내 휴대폰으로 보내 직접 열어봅니다. 글자가 너무 작지 않은지, 버튼이 손가락으로 누르기 쉬운지 확인합니다.

2. 시크릿 창에서 열기

로그인된 교사 계정의 도움 없이도 열리는지 확인합니다. Chrome의 시크릿 창이나 다른 브라우저에서 주소를 붙여넣어 봅니다.

3. 학생 계정으로 열기

학교 Google 계정이나 학생용 테스트 계정으로 접속합니다. Drive, Forms, Sites 자료가 학생 권한에서도 열리는지 봅니다.

4. 핵심 버튼만 먼저 누르기

모든 링크를 한 번에 확인하기 어렵다면 가장 중요한 버튼부터 누릅니다. 제출, 자료 보기, 활동 시작, 다음 화면 이동 버튼은 반드시 확인합니다.

5. 학교 밖 화면도 확인하기

가정에서도 열어야 하는 자료라면 학교 와이파이 밖에서도 확인합니다. 학교 내부망이나 계정 제한 때문에 막히는 자료가 있는지 봅니다.

6. 응답 저장까지 끝까지 해보기

Forms, Tally, AppSheet, Supabase, Firebase를 연결했다면 테스트 응답을 하나 제출합니다. 교사가 확인해야 할 위치에 실제로 저장되는지 확인합니다.

학생 화면에서 자주 생기는 문제 자료가 잠기거나 이미지가 빠질 때 먼저 확인합니다.
사이트는 열리는데 Drive 자료가 안 열려요웹페이지 공개 범위와 Drive 파일 공개 범위는 서로 다릅니다.Drive 파일에서 링크 공유 또는 학교 계정 권한을 따로 확인합니다.
Forms 제출이 안 돼요응답 받기가 꺼져 있거나, 학교 계정 로그인만 허용되어 있을 수 있습니다.학생 계정으로 열어보고 응답 권한과 제출 후 화면을 확인합니다.
Canva·Genially 링크가 교사에게만 보여요편집 링크를 공유했거나 공개 링크가 아닌 경우가 많습니다.학생에게 보낼 링크가 보기용·게시용 링크인지 확인합니다.
Netlify 주소는 열리는데 이미지가 안 보여요이미지 폴더가 함께 올라가지 않았거나, 파일 이름의 대소문자가 다를 수 있습니다.폴더 전체를 다시 올리고 이미지 경로를 확인합니다.
학생 휴대폰에서 화면이 깨져요카드 폭, 표, 긴 버튼 문구가 모바일 화면보다 넓을 수 있습니다.AI에게 “모바일에서 한 줄씩 내려오게 수정해줘”라고 요청합니다.
교사가 바로 고쳐볼 수 있는 문제 링크, 권한, 이미지, 모바일 버튼을 먼저 봅니다.
먼저 이 표의 순서대로 점검하세요 아래 문제는 대부분 교사가 공유 설정이나 파일을 다시 확인하면 해결할 수 있습니다. 복잡한 기술 문제로 보기 전에, 내가 바꿀 수 있는 부분부터 확인합니다.
링크를 잘못 보냈을 때편집 주소, 미리보기 주소, 게시 주소가 다를 수 있습니다.학생에게 보낼 주소를 시크릿 창에서 열어보고 다시 복사합니다.
Drive 파일이 잠겼을 때문서, 슬라이드, PDF 파일의 공유 권한을 따로 엽니다.사이트가 열려도 Drive 파일은 별도 잠금이 걸릴 수 있습니다.
Forms 응답이 막혔을 때응답 받기 상태, 로그인 필요 여부, 제출 횟수 제한을 확인합니다.테스트 응답을 하나 제출한 뒤 교사용 응답 화면에서 확인합니다.
이미지가 보이지 않을 때이미지 파일을 함께 올렸는지, 파일 이름과 폴더 이름이 정확한지 확인합니다.대소문자와 띄어쓰기가 다르면 배포 후 이미지가 빠질 수 있습니다.
모바일에서 버튼이 눌리기 어려울 때버튼 문구를 줄이고, 버튼 사이 간격을 넓힙니다.표가 길면 카드형 목록으로 바꾸는 것도 좋습니다.

서비스 권한·자동화 점검하기

교사가 웹페이지를 자유롭게 만들려면 어떤 서비스에 무엇을 맡기는지도 알아야 합니다. 무료 플랜, 권한, 외부 서비스 연결, 자동화 기능은 수업 전에 한 번 더 확인하는 것이 안전합니다.

금액과 무료 조건

가격은 사용 직전에 확인

  • 무료로 시작 가능한지 확인
  • 트래픽, 저장 용량, AI 기능, 팀 기능 제한 확인
  • 가격과 무료 조건은 바뀔 수 있으므로 공식 사이트에서 재확인
구글 권한

Drive와 Sites 권한 맞추기

  • 사이트 공개 범위와 Drive 파일 공개 범위가 일치하는지 확인
  • 구글 폼, 문서, 슬라이드가 학생 계정에서 열리는지 확인
  • 학교 계정만 허용할지 링크가 있는 모두에게 열지 결정
Apps Script

자동화는 권한부터 점검

  • 웹앱 실행 권한이 누구에게 열려 있는지 확인
  • 학생 개인정보를 Google Sheets에 저장하는지 확인
  • 자동 이메일이나 조회 기능은 테스트 계정으로 먼저 점검
응답 저장 도구

외부 서비스 저장 위치 확인

  • Google Forms, Tally, AppSheet, Supabase, Firebase 중어디에 응답이 저장되는지 확인
  • 이름, 학번, 연락처, 얼굴 사진은 꼭 필요한 경우가 아니면 받지 않기
  • 응답을 누가 볼 수 있는지, 언제 삭제할지, 공개 결과에 개인 응답이 드러나는지 점검
자유화의 핵심 교사의 웹페이지 제작 자유화는 플랫폼을 많이 쓰는 일이 아닙니다. 수업 목적에 맞게 필요한 도구를 선택하고 안전하게 연결하는 능력입니다.
사용한 서비스가 없다면 건너뛰세요 아래 표는 모든 교사가 전부 확인해야 하는 목록이 아닙니다. 예를 들어 Canva 웹페이지와 Forms만 썼다면 Canva, Forms, 학생 기기 확인만 먼저 보면 됩니다. Supabase나 Firebase를 쓰지 않았다면 해당 항목은 지금 단계에서 읽지 않아도 됩니다.
서비스별 권한 확인 미니 체크 내가 사용한 서비스만 골라 확인합니다.

사용한 서비스만 골라 확인하세요

웹페이지 주소가 열려도 그 안에 연결한 자료는 따로 잠겨 있을 수 있습니다. 학생에게 보내기 전에 사용하는 서비스별로 아래 항목만 빠르게 확인하세요.

Google Drive문서, 슬라이드, PDF, 이미지 파일의 공유 범위를 확인합니다.학생이 학교 계정으로 열어야 하는지, 링크가 있는 사용자에게 열지 정합니다.
Google Forms응답 받기가 켜져 있는지 확인합니다.로그인 필요, 이메일 수집, 1회 응답 제한을 봅니다.제출 후 학생에게 보이는 메시지도 확인합니다.
Google Sites사이트 게시 범위와 삽입한 Drive 자료의 공개 범위를 따로 확인합니다.게시 후 보기 주소를 학생 계정이나 시크릿 창에서 다시 엽니다.
Canva편집 링크가 아니라 보기용 또는 웹사이트 게시 링크를 공유합니다.Canva Sheets나 삽입 자료에 개인정보가 들어가지 않았는지도 확인합니다.
Genially공개 또는 공유 설정이 학생에게 열려 있는지 확인합니다.팝업, 버튼, 다음 화면 이동이 모바일에서도 작동하는지 봅니다.
Netlify폴더 전체가 올라갔는지 확인합니다.이미지, CSS, PDF 파일이 빠지지 않았는지 배포 주소에서 다시 봅니다.
GitHub PagesPublic 저장소에 학생 정보나 비밀 키가 올라가지 않았는지 확인합니다.Pages 설정에서 공개할 브랜치와 폴더가 맞는지 봅니다.
