AI로 만드는 법

AI와 대화하며 교육용 웹페이지 만들기

이 페이지는 AI에게 어떻게 말하면 좋을지 다룹니다. index.html 초안 만들기, 수정하기, 공개 전 점검하기를 차례대로 봅니다. 웹페이지가 정말 필요한지는 처음 시작 페이지에서 먼저 판단하세요.

이 페이지 목차
1. 필수 안내 확인하기 2. AI에게 넣지 말아야 할 정보 3. 바이브코딩 방식 이해하기 4. index.html 초안 만들기 5. 사례 기반 프롬프트 활용하기 6. 기존 사이트 수정·보강하기 7. 프롬프트 모음 활용하기
필수 안내 확인하기 AI로 웹페이지를 만들기 전에 처음 시작 페이지에서 “정말 필요한 상황인지” 먼저 판단하세요. 웹페이지 제작은 필수가 아니라, 수업 흐름을 더 명확하게 만들 때 선택하는 도구입니다. 공지·과제·성적 관리는 Classroom/LMS가 더 빠를 수 있습니다. 설문은 Forms/Tally, 정형 퀴즈는 Kahoot/Quizizz/Wordwall이 더 나을 수 있습니다. AI 활용은 기존 앱으로 부족한 화면이나 흐름을 만들 때 쓰는 보조 수단으로 보는 것이 안전합니다.

AI에게 넣지 말아야 할 정보

AI는 제작을 도와주는 훌륭한 조력자입니다. 하지만 실제 학생 정보나 비밀 설정값을 그대로 넣는 곳은 아닙니다. 질문할 때는 예시 데이터로 바꿔서 물어보는 습관이 필요합니다.

개인정보

실제 학생 정보를 붙여넣지 않기

학생 이름, 학번, 성적, 상담 내용은 AI에게 그대로 넣지 않습니다. 얼굴 사진이나 연락처도 실제 자료 대신 예시로 바꿔 질문합니다.

대신 “학생A”, “1모둠”, “예시 응답”처럼 바꿔 질문합니다.

비밀 키

관리자 키와 비밀 API 키 넣지 않기

Supabase service role key는 그대로 넣지 않습니다. 외부 API 비밀 키나 서버 관리자 키도 코드와 대화창에 붙여넣지 않습니다.

키 종류를 모르면 “이 키를 브라우저 코드에 넣어도 되는지 판단해줘”라고 먼저 물어봅니다.

비공개 자료

학교 내부 링크와 문서 권한 확인하기

비공개 Drive 문서, 내부 회의 자료, 학생 결과물 링크는 먼저 공개 범위를 확인합니다. AI에게 질문할 때는 예시 링크로 바꿔서 설명합니다.

AI가 만든 페이지에 실제 링크를 넣기 전 학생 계정에서 열리는지 확인합니다.

코드를 받은 뒤 다시 물어볼 말 
방금 만든 웹페이지 코드를 공개하기 전에 점검해줘.

1. 학생 개인정보가 코드나 화면에 들어갔는지
2. 공개하면 안 되는 링크나 문구가 있는지
3. API key나 비밀 키가 브라우저 코드에 들어갔는지
4. 모바일 화면에서 버튼과 글자가 적절한지
5. Google Forms, Tally, Drive 링크 권한을 따로 확인해야 하는지
초보 교사 기준으로 체크리스트를 만들어줘.

바이브코딩 방식 이해하기

바이브코딩은 코드를 처음부터 외워서 쓰는 방식이 아닙니다. 만들고 싶은 결과물을 AI에게 말로 설명하고, 대화하며 수정하는 방식입니다.

교사가 하는 일

  • 수업의 목적과 대상을 설명합니다.
  • 학생이 어떤 순서로 활동해야 하는지 알려줍니다.
  • 활동지, 영상, 제출 링크 등 필요한 자료를 정리합니다.
  • 모바일에서 보기 편하게, 버튼을 크게 같은 조건을 말합니다.

AI가 도와주는 일

  • HTML, CSS, JavaScript 초안을 만듭니다.
  • 문구를 더 친절하게 다듬습니다.
  • 버튼, 카드, 탭 같은 화면 구성을 제안합니다.
  • 모바일 화면과 접근성 점검을 도와줍니다.

index.html 초안 만들기

Netlify, GitHub Pages, Cloudflare Pages처럼 HTML 파일을 올리는 방식은 먼저 index.html 파일이 필요합니다. 아래 프롬프트를 복사해 수업 내용만 바꿔보세요.

AI에게 주면 좋은 정보

  • 페이지 제목: 예) 우리 동네 인구 변화 분석
  • 대상: 예) 고등학교 1학년 통합사회
  • 목적: 예) 지역 데이터를 읽고 해석하기
  • 활동 순서: 자료 읽기, 분석하기, 발표하기, 제출하기
  • 자료 링크: 활동지, 영상, 제출 폼
  • 디자인 조건: 모바일 친화적, 큰 버튼, 차분한 색감
교육용 웹페이지를 만들고 싶어.

다음 조건으로 index.html 파일 하나에 들어갈 코드를 만들어줘.

기존 사이트 수정·보강하기

이미 만든 웹사이트가 있다면 처음부터 다시 만들 필요가 없습니다. Codex처럼 파일을 직접 읽고 수정할 수 있는 AI에게 요청할 수 있습니다. 예: “기존 구조를 유지하면서 필요한 부분만 보강해줘.”

기존 구조 유지

디자인을 갈아엎지 않기

이미 만든 메뉴, 카드, 탭, 색상 체계를 유지합니다. 필요한 설명만 늘리도록 요청합니다.

도구 설명 추가

수정 후 점검

링크와 탭을 확인하기

내비게이션 링크, 목차 이동, 탭 전환을 확인하도록 요청합니다. 모바일 카드 길이와 개인정보 주의 문구 연결도 함께 점검합니다.

Codex에게 줄 수 있는 프롬프트 예시

중요 조건:
- 기존 디자인과 CSS 클래스를 최대한 유지해줘.

  choose.html은 도구 선택 기준,
  tools.html은 프로그램별 실제 사용법,

1. 모든 메뉴 링크가 열리는지
2. 목차 링크가 맞는 위치로 이동하는지
3. tools.html의 탭 전환이 작동하는지
4. 모바일 화면에서 카드 내용이 너무 길어지지 않는지
5. 공개 전 점검 페이지와 개인정보 안내가 연결되는지
